```
(define (p) (p))
(define (test x y) (if (= x 0) x y))
(test 0 (p)) ;を評価
```

## 適用順評価の場合
これは実際の処理系で用いられる評価順序であり，引数をその都度評価してから展開していくものである．
今回の場合，
```
(p)
```
を評価すると無限ループに陥るのだが，
適用準評価の場合は，
```
(test 0 (p))
```
を展開する前に
```
(p)
```
が評価されるため無限ループとなる．

## 正規順評価の場合
この評価順序では，展開できるところまで行ってから，評価を行う．
今回の場合，
```
(test 0 (p))
↓
(if (= 0 0) 0 (p))
↓
0
```
となり，
```
(p)
```
を評価することなく計算を終えることとなる．
